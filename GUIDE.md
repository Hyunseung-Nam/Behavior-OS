# Behavior OS 사용 설명서

---

## 앱의 목적

이 앱은 투두 앱이 아닙니다.

**해야 할 순간에 실제로 행동하도록 돕는 시스템**입니다.
잊고 있다가 알림을 보고 생각나는 것, FocusPage를 열었을 때 지금 해야 할 것이 눈에 들어오는 것이 핵심입니다.

---

## 화면 구성

### 1. FocusPage (메인 화면)

앱을 열면 가장 먼저 보이는 화면.

```
BEHAVIOR OS                              [목록]

           [경과 시간]

           [일정 제목]

        "왜 중요한가"

    [ 최소한 이것만 ]

     상황에 맞는 메시지

        [  행동 완료  ]
        [   10분 연기  ]
```

- **지금 해야 할 일정이 없을 때**: "지금 당장 할 일이 없습니다." + 일정 등록 버튼
- **일정이 있을 때**: 카테고리 컬러로 포인트 색상이 바뀌며 일정 정보 표시

---

### 2. 일정 목록 (`/schedule`)

FocusPage 우측 상단 목록 아이콘으로 진입.

- 현재 등록된 모든 일정 확인
- 상태 배지: `대기` / `진행중` / `연기됨`
- **수정**: 항목을 탭 → 수정 폼 열림
- **삭제**: 항목을 왼쪽으로 스와이프 → 삭제 확인
- **등록**: 우측 하단 `+` 버튼
- **루틴 설정**: AppBar 반복 아이콘
- **완료 기록**: AppBar 시계 아이콘
- **놓친 일정**: AppBar 받은편지함 아이콘

---

### 3. 일정 등록/수정 폼

`+` 버튼을 누르면 등록 모드로, 항목을 탭하면 수정 모드로 바텀시트가 올라옵니다.

| 항목 | 필수 여부 | 설명 |
|------|-----------|------|
| 카테고리 | 선택 | 공부 / 운동 / 업무 / 휴식 / 기타 — FocusPage 컬러 결정 |
| 무엇을 해야 하나요? | **필수** | 일정 제목 |
| 왜 중요한가요? | 선택 | FocusPage에 표시됨. 내가 쓴 이유가 보임 |
| 최소한 이것만 | 선택 | FocusPage에 표시됨. 시작 마찰을 낮추는 용도 |
| 날짜 / 시간 | **필수** | 일정 예정 시각 |

**팁**: "왜 중요한가요?"와 "최소한 이것만"을 채울수록 FocusPage 경험이 풍부해집니다.

---

### 4. 보관함 (`/missed`)

일정 목록 받은편지함 아이콘으로 진입.

놓친 일정(scheduledAt + 60분 초과)이 자동으로 여기로 이동됩니다.

- **재스케줄**: 날짜/시간을 새로 잡아 다시 등록 (캘린더 이벤트도 자동으로 교체)
- **삭제**: 스와이프로 제거

---

### 5. 완료 기록 (`/history`)

일정 목록 시계 아이콘으로 진입.

완료된 일정을 날짜별로 모아서 보여줍니다.

- 예정 시각 vs 실제 완료 시각 비교
- **정시** (5분 이내): 초록색
- **지연** (30분 이내): 주황색
- **지연** (30분 초과): 빨간색

---

### 6. 루틴 설정 (`/settings/routines`)

일정 목록 반복 아이콘으로 진입.

매일 반복 알림 항목을 관리합니다.

- **추가**: AppBar `+` 버튼 → 제목 + 알림 시각 입력
- **수정**: 항목 탭
- **삭제**: 왼쪽으로 스와이프
- **on/off 토글**: 우측 스위치 — 비활성화 시 알림 취소, 재활성화 시 즉시 재등록

---

## 알림 흐름

일정을 등록하면 자동으로 알림이 설정됩니다.

```
[예정 시각 30분 전]  →  "30분 후 예정" 알림
[예정 시각 10분 전]  →  "10분 후 시작" 알림
[예정 시각 정각]     →  "지금 바로 시작하세요" 알림
[정각 후 5분 간격]   →  Nagging 알림 최대 12회 (완료/연기하지 않은 경우)
```

**루틴 알림**: 설정한 시각에 매일 반복 알림이 옵니다. 일정 알림과 별개로 동작합니다.

알림을 탭하면 앱이 열립니다.

> **첫 실행 시**: iOS가 알림 권한 허용 창을 자동으로 띄웁니다. **허용**을 눌러야 알림이 동작합니다.
> 허용 후 설정 → 알림 → behavior_os → 배너 스타일을 **지속**으로 바꾸면 배너가 사라지지 않습니다.

## FocusPage 메시지

경과 시간은 단순 레이블로만 표시됩니다 (`N분 경과`).

메시지는 **본인이 등록 시 입력한 "왜 중요한가요?"** 그대로 표시됩니다.
앱이 만든 문구가 아니라 과거의 내가 지금의 나에게 하는 말.

---

## 버튼 동작

| 버튼 | 동작 |
|------|------|
| **행동 완료** | 일정을 완료 처리. 모든 알림 취소. FocusPage에서 사라짐 |
| **10분 연기** | Nagging 알림 취소. 10분 후 다시 알림. 상태 → snoozed |

---

## 카테고리 컬러

| 카테고리 | 컬러 |
|----------|------|
| 공부 | 파란색 |
| 운동 | 주황색 |
| 업무 | 보라색 |
| 휴식 | 초록색 |
| 기타 | 회색 |

---

## 상태 흐름

```
등록 → pending
예정 시각 도래 → active (FocusPage에 표시됨)
행동 완료 → completed (목록에서 사라짐)
10분 연기 → snoozed
60분 초과 → missed (보관함으로 이동)
```

---

## 캘린더 동기화

일정을 등록하면 iOS 네이티브 캘린더에 자동으로 추가됩니다.

```
앱에서 일정 등록
    ↓
iOS 캘린더 권한 요청 (최초 1회)
    ↓
"Behavior OS" 전용 캘린더 생성 (없을 경우)
    ↓
제목 + 시각 이벤트 추가
```

- **목적**: 다른 일정과 겹치는지 캘린더 앱에서 한눈에 확인
- **동기화 범위**: 제목 + 시작 시각만 (`why`, `최소한 이것만`은 앱 전용)
- **삭제**: 앱에서 일정을 삭제하면 캘린더 이벤트도 함께 제거
- **재스케줄**: 시각 변경 시 기존 이벤트 삭제 후 새 이벤트 추가

> **첫 실행 시**: 캘린더 권한 허용 창이 뜹니다. **허용**을 눌러야 동기화가 동작합니다.

## macOS vs iOS 차이

| 기능 | macOS | iOS |
|------|-------|-----|
| 일정 등록/조회 | O | O |
| 알림 | X (테스트 환경 미지원) | O |
| 백그라운드 상태 체크 | X | O |
| 캘린더 동기화 | X | O |

macOS는 UI 확인 및 데이터 입력 테스트 용도로만 사용하세요.
실제 알림과 캘린더 동기화는 iPhone에서 실행해야 동작합니다.
